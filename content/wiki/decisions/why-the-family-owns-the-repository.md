# Why the family owns the repository.

The wiki is plain text in a git repository the family owns. Not Atrium's repository. Not a hosted SaaS. Not a folder inside someone else's account.

If Atrium ceases to exist tomorrow, the family's institutional memory continues to exist tomorrow.

This is not a marketing line. It is the operative architectural commitment that makes a buy decision rational for a UHNW principal. The downside case has to be bounded.

The mechanics are straightforward. At the start of every engagement, Atrium provisions a private GitHub repository in an organization the family controls. Operators are added as collaborators; the principal owns the org. Every commit is signed. Every change is auditable. The family can `git clone` the repo to a laptop, to a thumb drive, to a hardware security module — the memory is portable in a way that a SaaS service is structurally not.

If a family ever chooses to end the Atrium relationship, the repo is unchanged in their hands the day after the engagement closes. The schema, the decisions, the council resolutions, the founder's reasoning — all of it is there, in plain markdown, readable by any text editor, ready to be tended by a successor practice or by the family's own internal operator.

The reversal cost is the cost of archiving one folder.

This is the posture that makes the relationship a stewardship, not a dependency. Atrium earns its place by being useful, not by being inescapable.
